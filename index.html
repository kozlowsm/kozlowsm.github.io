The Twitter API

By: Michael Kozlowski

Welcome

Welcome to the Twitter API how-to guide!

This guide will teach you to get started with the Twitter API by buidling Twitter bots. This guide assumes you have a general understanding of JavaScript as well as JSON(JavaScript Object Notation).

Reference material for this guide include:

  • Twitter Developer Documentation
  • Twit - Twitter API Client

What is an API?

An API or Application Programming Interface, is, by definition, a set of routines, protocols, and tools for building software applications. This is a bit too technical, so lets break it down.

An API is usually released by a software company to accompany the product that they offer. The API is released to the public and it allows other software developers to design products that are powered by the same service. Essentially, an API gives you access to things that are happening behind the scenes without actually having to go behind the scenes. We aren't all that concerned with the inner workings of Twitter, we just want access to specific data within their system so we can build applications.

What are we going to learn?

  • We are going to learn how to extract useful information from the Twitter API by using the Twit library, an API client for NodeJS.

  • We are going to learn about the various methods by which we can obtain data from Twitter, including how to get information from tweets and users, how to post tweets using the API, and how to reply to users

  • We will put all of this information together at the end to build a Twitter Bot that responds to users when mentioned and sends them a motivational/positive message

Let's get started!

Getting Started

What you'll need:

Before we can get started digging into the Twitter API and building bots, we'll need to setup a few things first to ensure it all runs smoothly

  • We need to make sure NodeJs is installed
  • Create a new Twitter account for your bot
  • Create a new Twitter application
  • Obtain the API keys to access the API

Installing NodeJS

If you don't already have NodeJs installed on your computer, you can head over to NodeJs.org and follow the instructions for installing it on your system.

Setting up a Twitter account for our bot

Before we begin coding and interacting with Twitter, we will need to create a Twitter account for our bot. To do this head over to twitter.com/signup. Enter the the required infomration and pick a Twitter handle(name) for your bot.

Next, in order to communciate with the Twitter API we need to obtain our API keys. To do this, go to apps.twitter.com and click on Create an Application follow the directions on the screen that look something like this:

Now that we have created our application, we need to obtain our API keys. Head over to the "Keys and Access tokens" tab at the top of the page, once you are on this page you will need to scroll down and and click "Create my access token" after you have completed these steps the page you are on should like this:

Keep this page open, or take note of those keys, will need to use them later. You should see:

  • Consumer Key
  • Consumer Secret
  • Access Token
  • Acces Token Secret

Setting up our local environment

This is the last step before we get into using the Twitter API so bear with me.

  • Create a new directory or folder to save the code for the Twitter bot, I named mine "Twitter Bot"

  • Now head over to the terminal/command line and use the cd command to go into the newly created Twitter Bot directory

  • In this directory we want to create a 'package.json' file - this file contains the metadata relevant to the project as well as the project dependencies. In the terminal enter npm init and follow the steps to create the file. *npm stands for Node Package Manager and it is used to install node package for specific projects

  • Now we want to install the Twit library that we mentioned earlier, this will allow us to access and interact with the Twitter API. In the terminal enter npm install twit --save

  • Finally, create the file you will be writing your code in, I named mine index.js

Now we are ready to start coding!

Twitter API Stream feature

Now we are going to talk about the stream feature of the Twitter API. We will then create a bot that takes advantage of this by responding to users who follow our bot.

The stream feature of the Twitter API gives us access to Twitters global stream of tweet data, this means we are able to get real time data tweet data. The two main streams available are the public stream and the User Stream.

  • Public Stream- This is a stream of all the public data flowing through twitter, this essentially gives us a view of every tweet happening on twitter.
  • User stream - This is a stream that provides data and events for a specific user, and the one we will be using. This is concerned with the data flowing into a single user, in this case, our bot.

We will use the 'User' stream and the follow event to get to get data when anyone follows our bot

Now to break it down:

  • Line 13: var stream = t.stream('user') Creates a variable called stream and uses the twit object to set the stream to 'user', this will get data related to our twitter account
  • Line 17: stream.on('follow', followResponse) this is turning the stream on so we can get real time data. 'follow' sets the event to listen for, when someone follows our account we will execute the function followResponse()
  • In the last few lines we are getting the JSON response and printing it to the console. We will look through this to find out how to get the name of the user we want to tweet a response to
